액션 태그/ 세션/ 쿠기

1. 액션테그
	jsp 페이지에서 다른 페이지에 접근할수 있도록 태그를 이용하여 구현된 기능

	- 웹페이지의 공통 부분을 모듈화
	- 페이지 흐름 제어 include forward
	- 파리미터 전송 param

웹 앱은 많은 페이지들의 서로 공통된 부부을 공통적으로 포함하고 있다.
이러한 부분을 부품화하여 재사용 하면 각각의 웹 페이지마다의 필요한
부분을 공뷰할 수 있겠된다.

따라서 코드를 단순화할수 있고 가독성을 향상시키도록 반복하여 코딩하지 않아도 되기 때문에 시간과 노력을 줄일 수 있다.
또 필요한 경우 유지보수를 용이하게 할 수 있다.

<jsp:include> 액션태그
실행처리 결과가 현재 jsp:include액션 태그 위치에 포함되어 컴파일 되어 실행된다.

flush 속성은 포함할 페이지를 싱행하기 전에 출력 버퍼를 지울지를 결정한다.
flush 속성이 true(기본값) 이면 포함할 페이지를 실행하기 전에 출력 버퍼를
지우고 페이지 실행 결과가 현재 페이지 위치에 포함되고  false 이면 출력 버퍼를
비우지 않고 포함될 페이지 실행 결과가 현재 위치에 포함된다.


파라미터 전송
웹 페이지의 공통 스크립트 요소에 특정 파라미터를 전달할수 있다. 
jsp:include 액션 태그 내부에 jsp:param 액션 태그를 포한하는 형태로
사용한다. 


include 지시문
include 지시문은 include 액션태그와 같이 서로 다른 웹 페이지에서 공통으로
사용되는 페이지를 포함시켜 사용한다.
주로 header 또는 footer 메뉴 들을 모듈화 하여 재사용하기 위해 사용된다.
include 액션태그와는 달리 단순하게 포함될 내용이 현재 페이지에 텍스트로 포함되어 사용된다는
차이점이 있다.

따라서 페이지 내의 변수가 공유되기 때문에 변수 중복오류 발생에 주의해야 한다.

include 액션태그			include 지시문
동적						정적
일부 기능을 모듈화 			코드나 문장을 모듈화
호출할때마다 재 컴파일			최초 한번만 처리
처리 결과가 포함				코드 자체가 포함
호출 페이지로 이동후 다시 복귀		이동없음

사용은 include지시문을 주로 사용한다.


jsp:forward 액션태그
현재 페이지에서 다른 페이지로 이동할 경우에 주로 사용

include 태그와 동작방식은 동일하지만 약간 다르게 동작한다. forward 되는 페이지가 
생성한 응답 결과를 웹 브라우저에게 전달한다는 점이 다르다.
호출한 페이지는 출력버퍼에서 비원진다.
즉, jsp:forward 액션태그의 이전 내용은 출력 버퍼에서 사라지고 이후 실행조차 되지 않는다. 

현재 jsp 페이지에서 include 액션태그를 만나면 include 액션태그에서 page속성으로 지정한
외부 파일의 실행 내용이 현재 jsp 페이지의 출력 버퍼에 추가 저장되어 출력된다.

그리고 include 액션태그는 외부 파일이 실행된 후 현재 jsp 페이지로 실행 제어를 리턴한다.


sendReddirect() 와의 차이점.
forward 액션태그는 이동 url페이지의 내용으로 변경되지만 브라우저 주소창은 변하지 않는다.


2. cookie

연결과 상태유지 특성에 대한 문제점을 해결하는 방법 중 하나이다.
쿠키를 사용하여 클라이언트의 상태 정보를 웹 서버로 전송할 수 있으며 전송받은 서버는 특정
클라이언트인지 여부를 확인 할수 있다.

헤더에 포함되어 전달된다.

클라이언트가 브라우저를 통해 서버에게 서비스를 요청하면 웹서버는 헤더에 포함된 쿠키 정보를 확인하게 된다.

이 생성된 쿠키 정보는 클라이언트에 저장된다.
쿠키가 삭제 되기 전까지 또는 브라우저를 종료하더라고 정해진 유효시간 이전에는 계속해서 웹 서버에 서비스를 요청할때마다
헤더에 포함되어 전송된다.
